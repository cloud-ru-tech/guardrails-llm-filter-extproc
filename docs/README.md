## Коротко о сервисе

Сервис — это Envoy **external processor** (ext_proc v3, двунаправленный gRPC) в
data-path LLM-шлюза. На запросе он сканирует JSON-тела OpenAI/Anthropic набором
из ~260 regex-правил и заменяет найденные значения плейсхолдерами (`<EMAIL_1>`);
на ответе восстанавливает оригиналы — в том числе внутри SSE-потоков и аргументов
tool-call. LLM никогда не видит чувствительные значения, клиент никогда не видит
плейсхолдеры.

Каждое изменение подчиняется двум инвариантам: модель вердикта — **только
маскировать/пропускать** (запрос никогда не блокируется), а data-path работает
**fail-open** (любая внутренняя ошибка — сбой хранилища, ошибка маскирования,
некорректное правило — пропускает трафик, а не ломает его). Глобальная политика
живёт в подключаемом хранилище (`in_memory`/`redis`/`postgres`), меняется на лету
через management API и может быть только **сужена** для конкретного запроса
доверенным заголовком шлюза.
